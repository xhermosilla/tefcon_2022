Servicio rest

API REST con un único endpoint que permite buscar palabras sobre un fichero indicado por parámetro

Índice

  • Entornos de desarrollo integrado
  • Impresiones durante la codificación
  • Prueba de performance
  • Tamaño de las imágenes docker

Descripción general del procedimiento

  • Desarrollo de servicio rest para contar palabras en un fichero (1 por lenguaje)
  • Utilización de artillery para probar rendimiento
  • Mover el servicio a una imagen docker

En negrita el IDE seleccionado para realizar el desarrollo

  • VSCode

Python

  • PyCharm de JetBrains

  • PyDev

Impresiones durante la codificación

Notas sobre las impresiones durante la codificación en cada lenguaje del servicio rest

Nos ha gustado:

  • Rápido durante el desarrollo (sorprende pensar que es un lenguaje compilado)
  • No hay que lidiar con la gestión de memoria
  • Baja complejidad y código reducido (inferencia de tipos, etc)

No tanto:

  • Nos ha costado encontrar documentación y tutoriales
  • La forma de gestionar errores es "diferente"

Nos ha gustado:

  • Fácil de aprender
  • Código legible (mejora bastante respecto a Java)

No tanto:

  • Poca documentación específica de Kotlin (aunque si de java)
  • Complejidad para iniciar un proyecto (gestión de dependencias)

Nos ha gustado:

  • Sintaxis simple y sencilla
  • Facilidad para iniciar un proyecto
  • Muy productivo

No tanto:

  • Fácil cometer errores

Nos ha gustado:

  • Facilidad de uso y elegancia
  • Facilidad para trabajar con JSON

No tanto:
