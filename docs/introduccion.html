Puntos fuertes y débiles

¿En qué cosas destacan, para bien o para mal?

Pregunta:

¿Cuál crees que es más escalable?

Ver información

Go

Puntos fuertes: muy rápido, muy escalable

Puntos débiles: gestión de errores, dificultad inicial

Kotlin

Puntos fuertes: compatilibidad java, menos código, menos errores

Puntos débiles: comunidad pequeña, no tan maduro como Java

Node.js

Puntos fuertes: comunidad grande, prototipos simples, mismo lenguaje front y back

Puntos débiles: sólo una CPU, gestión de dependencias

Python

Puntos fuertes: comunidad grande, prototipos simples

Puntos débiles: velocidad, consumo memoria

Python, node, kotlin, go... fight!
