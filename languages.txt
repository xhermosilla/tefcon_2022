Python:
- Tipos dinámicos (estáticos con mypy o pyre)
- Garbage collection
- object-oriented, imperative, functional, structured, reflective
- Lanzado en 1991
- Companies using Python: Google, Intel, IBM, NASA, Pixar, Netflix, Facebook, JP Morgan Chase, Spotify, Reddit, Pinterest, and Instagram
- The language is used for:
    Back end development 
    Data science 
    App development
- Puntos fuertes: comunidad grande, prototipos simples
- Puntos débiles: velocidad, consumo memoria
- 2M preguntas en stackoverflow.com

Go:
- Tipos estáticos
- Garbage collection
- concurrent, imperative, object-oriented
- Lanzado en 2009
- Companies: Google, Uber, Twitch, Dailimotion, Sendgrid, Dropbox, SounCloud
- The language is used for:
    Cloud-based or Server-side apps 
    System programming
    Command-line tools
    Artificial intelligence and data science
- Puntos fuertes: muy rápido, muy escalable
- Puntos débiles: gestión de errores, dificultad inicial
- 66K preguntas en stackoverflow.com

Kotlin:
- Tipos estáticos
- Garbage collection (JVM)
- object-oriented, functional, imperative, block structured, declarative, generic, reflective, concurrent
- Lanzado 2011
- Apps using Kotlin: Slack, Netflix, Evernote, Basecamp, Blockbuster, Occaz, AYIO, MoveUP.
- Puntos fuertes: compatilibidad java, menos errores, menos código
- Puntos débiles: comunidad pequeña, no tan maduro como Java
- 80% of programmers use Kotlin mainly to build Android apps, 31% for back-end applications, and 30% for SDK/libraries
- 82K preguntas en stackoverflow.com (1.9M para Java)

node.js:
- Tipos dinámicos (estáticos con typescript)
- Garbage collection
- event-driven, functional, imperative, procedural, object-oriented
- Lanzado 2009
- Companies using: LinkedIn, Netflix, Uber, Trello, PayPal, NASA, eBay, Medium, Groupon, Walmart, Mozilla, GoDaddy
- Javascript is used for: 
    Web development
    Game development
    Mobile apps 
    Building web servers
- Puntos fuertes: comunidad grande, prototipos simples, mismo lenguaje front y back
- Puntos débiles: sólo una CPU, gestión de dependencias
- 450K preguntas en stackoverflow.com (2.5M para JavaScript)
